# 新人研修課題：ToDoリスト制作依頼書

**発行日:** 2026年4月23日  
**対象:** 新入社員研修参加者

---

## 1. 依頼概要

HTML・CSS・JavaScript を使ってブラウザ上で動く ToDoリストを作成し、GitHub Pages で公開してください。

本課題の目的は、以下の3点です。

- HTML / CSS / JavaScript の基礎を実際に手を動かして身につける
- Git・GitHub を使ったファイル管理と公開の流れを体験する
- ググる力・AIに聞く力を実践で鍛える

---

## 2. 使用技術

| 項目 | 内容 |
|---|---|
| 言語 | HTML / CSS / JavaScript |
| フレームワーク | 不使用（バニラJS のみ） |
| バージョン管理 | Git / GitHub |
| 公開方法 | GitHub Pages |

---

## 3. 必須要件

以下の機能をすべて実装してください。

### 機能要件

**タスクの追加**
- テキスト入力欄にタスク名を入力し、ボタンを押すと一覧に追加される
- 入力欄が空の状態でボタンを押しても追加されない

**タスクの削除**
- 各タスクに「削除」ボタンを設置し、押すとそのタスクが一覧から消える

**完了チェック**
- 各タスクにチェックボックスを設置し、チェックを入れると完了状態になる
- 完了状態のタスクは見た目（文字色・取り消し線など）で区別できる

### ファイル構成

以下の3ファイルで構成してください。

```
プロジェクトフォルダ/
├── index.html   （画面の構造）
├── style.css    （見た目・デザイン）
└── script.js    （動作・機能）
```

### GitHub 要件

- リポジトリを作成し、コードを push する
- GitHub Pages で公開し、ブラウザからアクセスできる URL を提出する
- コミットは**機能を1つ追加するごと**に行う（まとめて1回のコミットは不可）

## 4. 推奨する実装ステップ

以下の順番で進めることを推奨します。コミットのタイミングも記載しています。

| ステップ | 作業内容 | コミットメッセージ例 |
|---|---|---|
| 1 | `index.html` を作成し GitHub Pages で公開する | `first commit: ページを公開` |
| 2 | 入力欄とボタンを HTML に追加する | `add: 入力欄とボタンを追加` |
| 3 | ボタンを押したらタスクが追加される機能を実装する | `add: タスク追加機能を実装` |
| 4 | 削除ボタンを追加する | `add: タスク削除機能を実装` |
| 5 | 完了チェックボックスを追加する | `add: 完了チェック機能を実装` |
| 6 | CSS で見た目を整える | `style: デザインを調整` |

---

## 5. 情報収集のルール

詰まったときは、以下の順番で情報を探してください。

```
① まず自分で考える（5分）
      ↓
② Google で検索する（10〜15分）
      ↓
③ それでも解決しなければ AI（ChatGPT・Claude など）に聞く
      ↓
④ 最終手段として先輩・講師に質問する
```

**AI を使うときの注意事項**

AI が提示したコードをそのままコピペすることは禁止です。コードを使う場合は、1行ずつ読んで何をしているか自分の言葉で説明できる状態にしてから使ってください。コードレビューでは「なぜそのコードにしたか」を説明してもらいます。
